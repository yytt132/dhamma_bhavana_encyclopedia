# Support

Use GitHub Issues for project support.

## Good Issue Types

- Entry proposal.
- Source metadata correction.
- Copyright concern.
- Editorial review request.
- Website/tooling bug.
- Schema improvement.

## Before Opening an Issue

Please check:

- `README.md`
- `CONTRIBUTING.md`
- `docs/editorial-guide.md`
- `docs/copyright-policy.md`
- existing issues

## Copyright Concerns

If you believe material in the repository should not be public, open an issue with:

- file path
- reason
- source or rights concern

Maintainers should remove or quarantine disputed material while reviewing it.
