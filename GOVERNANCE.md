# Governance

The Dhamma Bhavana Encyclopedia is currently maintained by the founding maintainer.

As the project grows, governance should remain simple, transparent, and review-oriented.

## Roles

### Maintainer

Maintainers can:

- Merge pull requests.
- Set release milestones.
- Update project policies.
- Decide entry publication status.
- Remove copyright-risk material.
- Coordinate reviewers.

### Editor

Editors can:

- Review article structure and clarity.
- Suggest revisions.
- Check status changes.
- Improve Korean and English prose.

### Subject Reviewer

Subject reviewers can:

- Review doctrinal, historical, linguistic, or practice claims.
- Flag tradition-specific issues.
- Recommend source improvements.

### Contributor

Contributors can:

- Propose entries.
- Improve dossiers.
- Add source metadata.
- Draft glossary or comparison entries.
- Improve code and validation tools.

## Decision Principles

- Copyright safety comes first.
- Source-aware precision comes before speed.
- Do not flatten different traditions into one universal vocabulary.
- Human review is required for publication-quality status.
- Uncertainty should be marked rather than hidden.

## Status Changes

Only maintainers or delegated editors should mark an entry as:

- `reviewed`
- `published`

AI-generated drafts should remain `draft` or `sourced` until human review is complete.
