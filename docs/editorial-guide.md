# Editorial Guide

This guide defines how to write SEP-style entries for the project.

## Entry Shape

Long-form entries should follow this general structure when appropriate:

1. Introduction.
2. Term and translation issues.
3. Textual and historical background.
4. Philosophical significance.
5. Practice significance.
6. Interpretations across traditions.
7. Modern usage and misunderstandings.
8. Major debates.
9. Related concepts and distinctions.
10. Conclusion.
11. Glossary of selected terms.
12. Bibliography.
13. Related entries.
14. Supplements.

Not every entry needs every section. The outline should serve the concept.

## Claim Types

Mark claims using one or more of the following categories:

- `doctrinal`
- `practice-instruction`
- `phenomenological`
- `developmental-map`
- `historical-scholarly`
- `clinical-scientific`
- `self-help-popular`
- `esoteric-initiation`
- `editorial-interpretation`

## Evidence Status

Use evidence status to avoid false certainty:

- `primary-source`
- `traditional-commentary`
- `scholarly-secondary`
- `dictionary-reference`
- `clinical-study`
- `teacher-instruction`
- `memoir`
- `workbook`
- `popular-unsourced`
- `local-corpus-metadata`

## Language Rules

- Preserve original terms when they matter.
- Give language and script where known.
- Do not assume one English gloss equals one concept.
- Note variant spellings and transliterations.
- Link related terms instead of overloading one entry.

## AI-Assisted Drafting

AI may be used for:

- Dossier assembly.
- Outline proposals.
- First drafts.
- Cross-reference suggestions.
- Consistency checks.

Human review is required for:

- Interpretation of key doctrines.
- Translation choices.
- Sectarian or lineage claims.
- Safety-sensitive practice advice.
- Publication status changes.
