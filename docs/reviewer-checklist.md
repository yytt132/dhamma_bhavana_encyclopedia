# Reviewer Checklist

Use this checklist when reviewing entries.

## Basic Metadata

- [ ] Entry has `id`.
- [ ] Entry has `title`.
- [ ] Entry has `entry_type`.
- [ ] Entry has `status`.
- [ ] Entry has `summary`.
- [ ] Entry lists related entries.
- [ ] Entry lists source IDs.

## Editorial Quality

- [ ] Introduction explains why the concept matters.
- [ ] Original-language terms are treated carefully.
- [ ] Translation issues are clear.
- [ ] Tradition-specific meanings are not collapsed.
- [ ] Modern usage and misunderstandings are identified.
- [ ] Related concepts are distinguished.

## Source Awareness

- [ ] Claims are tied to source metadata.
- [ ] Claim types are clear.
- [ ] Evidence status is clear.
- [ ] Uncertainty is marked.
- [ ] Bibliography does not contain copied source text.

## Copyright Safety

- [ ] No copyrighted PDFs or EPUBs.
- [ ] No OCR dumps.
- [ ] No long copied passages.
- [ ] No long copied practice instructions.
- [ ] Public-domain or open-license material is labeled.

## Publication Status

- [ ] `stub` only contains placeholder or outline.
- [ ] `draft` may include unreviewed prose.
- [ ] `sourced` includes source metadata.
- [ ] `reviewed` has human review.
- [ ] `published` is stable and ready for public site rendering.
