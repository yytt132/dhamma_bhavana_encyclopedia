# API Credit Plan

API credits will be used for open-source maintenance and human-reviewed editorial workflows.

## Principles

- AI assists; humans decide.
- No copyrighted source text will be published.
- AI output must be reviewed before an entry becomes `reviewed` or `published`.
- Source metadata and provenance should remain visible.
- Korean-language accessibility is a core goal.

## Planned Uses

### 1. Concept Dossier Generation

For each concept, API credits can help create a structured dossier:

- Original terms and transliterations.
- Translation issues.
- Tradition-specific meanings.
- Related entries.
- Source candidates.
- Disambiguation notes.
- Open questions for human review.

### 2. SEP-Style Entry Outlines

Credits can help generate outline proposals for long-form entries:

- Introduction.
- Term and translation issues.
- Historical and textual background.
- Philosophical significance.
- Practice significance.
- Major debates.
- Related concepts.
- Bibliography candidates.

### 3. Source Metadata Normalization

Credits can help normalize public metadata:

- Title variants.
- Author names.
- Publication years.
- Tradition tags.
- Entry/source cross-references.

### 4. Cross-Reference Review

Credits can help check whether entries correctly link to related concepts:

- `bhavana`
- `samatha`
- `vipassana`
- `sati`
- `samadhi`
- `dhyana-jhana`

### 5. Korean-English Editorial Support

Credits can help produce and review bilingual materials:

- Korean summaries.
- English mission and contributor docs.
- Glossary term explanations.
- PR summaries.
- Issue triage notes.

### 6. Maintenance Automation

Credits can support:

- Issue classification.
- Pull request review summaries.
- Release note drafts.
- Copyright-risk review.
- Schema migration planning.

## Out of Scope

API credits will not be used to publish:

- Copyrighted PDFs.
- OCR dumps.
- Extracted book chapters.
- Long verbatim practice instructions from copyrighted sources.
